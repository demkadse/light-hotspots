1. Grundidee

Wofuer der Kalender gedacht ist

Der Light Hotspots Kalender ist eine Uebersichtsseite fuer Rollenspiel-Events auf dem Light-Datenzentrum. Er hilft dabei, aktuelle und kommende Termine schnell zu finden, nach Eventarten oder Orten zu filtern und alle wichtigen Informationen zu einer Veranstaltung an einem Ort zu sehen.

Wichtig zu wissen

Die Website dient hauptsaechlich der Darstellung. Das Erstellen, Bearbeiten und Absagen eigener Events geschieht ueber den Discord-Bot.

2. Website

Aufbau der Kalender-Website

Kopfbereich

Zeigt den Titel des Kalenders, einen kurzen Einleitungstext, den Discord-Button und den aktuellen Fokusbereich.

Filterbereich

Hier lassen sich Zeitfenster, Typen, Orte und die Sichtbarkeit von Events und Venues einstellen.

Tagesansicht

Jeder Tag des aktuellen Monats wird als eigene Station mit Event-Karten dargestellt.

Fussbereich

Enthaelt zusaetzliche Informationsbereiche wie Rechtshinweis, Transparenz und den Zugang zum Handbuch.

3. Website

Filter und Navigation

Zeitfenster

Die Schaltflaechen Alle Tage, Heute und Wochenende schraenken die angezeigten Tage ein.

Schnellnavigation

Je nach Situation bietet der Kalender Schnellziele wie Morgen, Monatsanfang oder Erstes Event.

Typ- und Ortsfilter

Nutzer koennen gezielt nach bestimmten Eventtypen oder Locations filtern. Zusaetzliche Schnellfilter-Chips helfen bei haeufigen Typen.

Events und Venues

Die Anzeige kann auf klassische Events, auf Venues oder auf beides gleichzeitig begrenzt werden.

Der Kalender merkt sich den zuletzt genutzten Zustand, darunter Filter, Kategorien und aktiven Tag.

4. Website

Event-Karten

Die Event-Karten sind die kompakte Schnellansicht eines Eintrags. Je nach verfuegbaren Daten koennen sie enthalten:

  • Bannerbild oder Platzhalterbild
  • Titel des Events
  • Kategorie und Typ
  • Uhrzeit
  • Ort und Server
  • Projektleitung
  • Kurzbeschreibung
  • Hinweis auf vorhandene Links

Wird kein Banner angegeben, verwendet die Seite automatisch das Platzhalterbild, damit jede Karte vollstaendig aussieht.

Abgesagte Events bleiben sichtbar, werden aber klar als abgesagt markiert.

5. Website

Detailansicht eines Events

Beim Anklicken einer Event-Karte oeffnet sich ein Detailfenster mit allen verfuegbaren Angaben.

Grunddaten

Titel, Kategorie, Typ, Datum, Uhrzeit, Ort, Server und Projektleitung.

Erweiterte Angaben

Wiederholung, Hinweise und Status bei abgesagten Events.

Links

Direkte Schaltflaechen fuer Discord-Server oder externe Informationsseiten, falls vorhanden.

Weitere Events

Falls passend, zeigt die Detailansicht weitere Veranstaltungen desselben Abends an.

6. Website

Zusatzbereiche auf der Website

Im Fussbereich der Seite befinden sich weitere Informationsfenster:

  • Rechtshinweis: Hinweise zur Plattform und zur konkreten Gestaltung
  • Transparenz: kurze Einordnung zur Entstehung des Projekts
  • Unterschiede Events/Venues: Hilfe bei der Einordnung der Eventarten
  • Handbuch: diese Dokumentationsseite
  • Datenschutzerklaerung: einfache Erklaerung dazu, welche Angaben fuer den Kalender noetig sind
7. Discord-Bot

So beginnt der Event-Prozess auf Discord

Im Discord-Server gibt es ein Event-Panel mit einer Start-Schaltflaeche. Nach dem Klick darauf sehen Nutzer normalerweise drei Wege:

  • Neues Event erstellen
  • Bestehendes Event bearbeiten
  • Veroeffentlichtes Event absagen

Damit lassen sich alle wichtigen Schritte fuer eigene Events direkt ueber die sichtbare Nutzerfuehrung erledigen.

8. Discord-Bot

Neues Event erstellen

Die Event-Erstellung ist als klarer 3-Schritte-Ablauf aufgebaut.

1

Basisdaten

Pflichtfelder sind Titel, Location, Datum, Uhrzeit und Beschreibung.

Formatregeln: Datum im Format TT.MM.JJJJ, Uhrzeit im Format HH:MM.

2

Auswahlmenues

Danach werden Kategorie, Typ, Server und Wiederholung ueber klare Dropdown-Menues ausgewaehlt. Dadurch muss so wenig wie moeglich frei eingetippt werden.

Ob ein Eintrag als Event oder Venue gefuehrt wird, entscheidet der Nutzer dabei selbst aktiv.

3

Zusatzangaben

Optional moeglich sind Endzeit, Projektleitung, Banner-URL, Discord-Link, externer Link und Hinweise.

Die Bild-URL muss auf eine Bilddatei zeigen, Discord-Links muessen zu discord.gg oder discord.com fuehren.

Vorschau nach jedem Schritt

Nach jedem Schritt zeigt der Bot eine Vorschau des aktuellen Events. So lassen sich Eingaben kontrollieren und bei Bedarf noch vor dem Absenden korrigieren.

9. Discord-Bot

Pruefung und Status eines Events

Nach dem Klick auf Jetzt zur Pruefung senden passiert aus Nutzersicht Folgendes:

  • Das Event wird zur Pruefung eingereicht.
  • Es ist noch nicht sofort oeffentlich sichtbar.
  • Nach der Pruefung gibt es eine Rueckmeldung.
  • Bei Freigabe wird das Event auf der Website sichtbar.
  • Bei Ablehnung wird ein Grund hinterlegt, damit das Event ueberarbeitet werden kann.

Moegliche Zustaende aus Nutzersicht

  • begonnen, aber noch nicht eingereicht
  • zur Pruefung gesendet
  • freigegeben und sichtbar
  • sichtbar, aber als abgesagt markiert
  • abgelehnt und zur Ueberarbeitung zurueckgegeben
10. Discord-Bot

Bestehendes Event bearbeiten

Ueber Bestehendes Event bearbeiten koennen Nutzer ihre vorhandenen Eintraege erneut oeffnen.

Der Bot zeigt dazu eine Auswahlliste mit den eigenen Events. Typischerweise sichtbar sind Titel, Datum und aktueller Status.

Nach der Auswahl wird das Event wieder im 3-Schritte-System geoeffnet. Der Ablauf ist derselbe wie bei einer Neuerstellung:

  • Basisdaten bearbeiten
  • Details bearbeiten
  • Extras bearbeiten
  • erneut zur Pruefung senden
11. Discord-Bot

Veroeffentlichtes Event absagen

Wenn ein bereits sichtbares Event nicht stattfinden kann, koennen Nutzer es ueber den Bot absagen lassen.

Auswahl

Der Bot zeigt alle eigenen Events, die bereits freigegeben wurden.

Bestaetigung

Vor dem Absagen fragt der Bot noch einmal nach, ob das Event wirklich als abgesagt markiert werden soll.

Wirkung

Das Event wird nicht geloescht, sondern bleibt sichtbar und wird klar als abgesagt markiert.

Sichtbarkeit

Besucher sehen weiterhin, dass das Event geplant war, aber nicht stattfindet.

12. Zusatzfunktion

Wochenfeed

Zusatzlich zum eigentlichen Kalender gibt es einen Wochenfeed fuer die naechsten sieben Tage.

  • Discord-Feed
  • RSS-Feed
  • JSON-Feed

Der Feed zeigt zuerst eine kompakte Wochenuebersicht und danach fuer jedes Event eine eigene Darstellung mit allen verfuegbaren Informationen.

13. Schnelluebersicht

Typischer Ablauf fuer Nutzer

  1. Kalender oeffnen und passende Events ansehen.
  2. Bei Bedarf ueber den Button zum Discord-Server wechseln.
  3. Im Discord den Event-Ablauf starten.
  4. Basisdaten, Details und Extras ausfuellen.
  5. Die Vorschau kontrollieren.
  6. Das Event zur Pruefung senden.
  7. Auf Rueckmeldung warten.
  8. Nach Freigabe das Event auf der Website wiederfinden.
  9. Spaeter bei Bedarf bearbeiten oder absagen.
Kurz gesagt

Die Website ist die gut lesbare Event-Uebersicht. Der Discord-Bot ist das Werkzeug fuer Einreichung, Bearbeitung und Absage.
